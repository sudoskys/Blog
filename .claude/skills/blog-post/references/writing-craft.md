## 无形过渡（不用过渡句实现段落衔接）

禁止模板化过渡句（"首先/其次/总之"、"接下来让我们看看"），但不能有 jarring 跳跃。
以下六种技术替代过渡句：

### 1. 回声词（Echo）

上一段的关键词自然出现在下一段开头，读者大脑自动建立联系。

```
...这种延迟最终导致了三次生产事故。
[新段] 第三次事故发生在凌晨两点。
```

### 2. 尾-头钩锁（Known → Unknown）

前一句末尾的信息成为下一句开头的信息。

```
...团队最终选择了事件驱动架构。
[新段] 事件驱动架构在他们的场景下意味着完全重写消息层。
```

### 3. 主题线（Thematic Thread）

贯穿全文的一个意象/类比串起各节。比如用「建筑」隐喻串联从地基到装修——
每节开头自然呼应这条线。

### 4. 问题桥（Question Bridge）

一节结尾自然引出读者此刻会想到的问题，下一节即是回答。
不写出问题本身——让下一节的内容隐含回答。

### 5. 时间/空间锚

通过场景转换、时间流动暗示过渡："三个月后，服务已经稳定运行了。"
比「接下来」自然得多。

### 6. 情绪级联

前一节结尾的情绪状态自然流入下一节起点。
一节以「发现」收尾，下一节以「带着这个发现去验证」开启。

**检验方法**：去掉标题后，纯文本仍然流畅 = 过渡成功。

## 视觉节奏

### 核心规则

- **每 300-400 词一次视觉断裂**（图片、列表、callout、表格、代码块均可）
- **密集元素后跟呼吸器**：大表格或密集代码块之后，放一个居中总结句或留白
- **段落 1-3 句**（手机上 4 行以上就是文字墙）
- **2-3 个代码块最优**——10+ 个代码块的文章反而表现差

### 各元素最佳用途

| 元素 | 最佳用途 | 反模式 |
|------|----------|--------|
| 截图 | 具体步骤、UI 操作、错误信息 | 未裁剪的整面 UI |
| 表格 | 并列比较（2-4 列，3-6 行） | 超 3 列手机崩溃 |
| Mermaid 图 | 架构、决策树、数据流 | 元素过多需要解码 |
| Callout/引用 | 关键结论、警告 | 过度使用导致什么都不突出 |
| 粗体 | 关键短语强调 | 过度加粗 → 层级崩塌 |

### Squint Test

把页面缩到看不清字。如果看起来像一堵墙，格式失败。
应该看到：深浅交替的块，有呼吸。

## 读者参与与完读率

### 硬数据

- 博客平均停留时间 < 90 秒
- 2000+ 字长文：仅 20-30% 读者到达末尾
- 79% 用户扫描而非逐字阅读
- 持续阅读 61 秒以上的帖子获得 15.6% 参与率（vs 3 秒内离开的 1.2%）

### 开头规则

- 引言 80-120 词最优（过短缺上下文，过长延迟价值承诺）
- 第一句 < 20 词
- 以「我」开头 → 比「How to」或「The」开头获得 3.2 倍参与
- 三种高效开头：戏剧性时刻、反常识统计、个人失败故事

### 保持滚动

- **重锚点**：约 500 词处、2000 词处、之后每 1000 词——放案例、截图、数据、反直觉观点
- **每节结尾暗示下一节的价值**（不是悬念，是自然引出）
- **每 300-500 词引入新观点或新问题**

## 观点文螺旋说服

CER 是单论点的论证框架。整篇文章的说服力还需要更大的结构。

### 螺旋深化

不是"论点→证据→结论"一次完成，而是同一论点在不同层面反复出现，每次更深一层。
第一次是现象观察，第二次是机制解释，第三次是边界条件。

### 2+1 框架（绕过读者心理防线）

读者看到「你应该做 X」时大脑立即寻找反驳理由。绕过方式：
1. 展示常见错误做法（解释为何失败）
2. 展示诱人的捷径（解释为何适得其反）
3. +1 正确路径——此时你的方案成为「唯一合理选择」

### 「同行者」姿态

呈现的不是「我知道很多」，而是「我和你站在同一困境里，刚蹚出一小段路」。
比「老师」姿态有效得多——中文读者对说教敏感。

### 信息颗粒度

知乎高赞的核心机制：「连续 14 天在 5:47-5:53 之间起床」比「建议早起」有 100 倍说服力。
权威感来自「我做过，且留了痕迹」，不来自「我知道很多」。

## 叙事节奏（短篇幅）

### 三模式切换

| 模式 | 功能 | 时间效果 | 3000 词中参考比例 |
|------|------|----------|------------------|
| 场景（Scene） | 沉浸感 | 放慢时间 | ~2200 词 |
| 摘要（Summary） | 压缩、移动 | 加速时间 | ~800 词 |
| 反思（Reflection） | 开出意义 | 暂停时间 | 穿插在场景中 |

### 速度变化 = 重要性信号

两句话总结角色的整个上午，然后在打开一封关键邮件的瞬间切换到实时——
速度转换本身告诉读者「注意，这里重要」。

### Speedy Narrative

介于场景与摘要之间：保持摘要的速度但锚定在具体细节。
即使在快速移动中也不放弃细节——不写"高中很快乐"，而是一句话里塞一个感官锚点。

### 场景选择（1500-5000 词博客）

- 选 3-5 个场景（不是更多）
- 标准：这个场景是否推进变化、提升 stakes、或揭示新信息？
- 用摘要桥接场景——摘要中也要有具体细节锚点
- 结尾落在一个有共振的音符上，不是「写到没有了就停」

### 节拍选择

| Tempo | 效果 | 适用 |
|-------|------|------|
| Presto | 能量、紧迫 | 多事件、时间跨度大 |
| Andante | 从容、探索 | 过程性叙述 |
| Adagio | 深度沉浸 | 聚焦单一时刻 |

叙事中心需要读者停留最久的地方用慢板，其余快板带过。

## 修订方法论

### 四轮修改法

| 轮次 | 聚焦 | 操作 |
|------|------|------|
| 1 | 结构 | 每节是否回答一个明确问题？删不推进论证的段落 |
| 2 | 表达 | 长句在哪里断气？「的」字数是否过多？ |
| 3 | 事实 | 数据来源验证、技术细节精确性、缩写词首次出现是否解释 |
| 4 | 读者体验 | 小白能否看懂？手机上段落是否太长？ |

### 去 AI 腔扩展自检（SKILL.md §5.3 的补充）

除了 SKILL.md 列出的禁用词和句式外，额外检查：

- [ ] 每段是否都整齐「开头-展开-总结」？→ 打破对称结构
- [ ] 连续数字列表超过 2 个？→ 改成叙述或混合形式
- [ ] 结尾是泛泛总结或展望未来？→ 换成具体行动/洞察/反问
- [ ] 缺少「我」的视角？→ 加入真实经历或判断
- [ ] 破折号是否过度使用？→ 改为逗号、句号或重排
- [ ] 排比三件套（X 是 Y，是 Z，更是 W）？→ 拆分为独立陈述
- [ ] 分号频率是否异常高？（博客语体中分号基线接近 0）
- [ ] 抽象主语可否换成具体主体？
